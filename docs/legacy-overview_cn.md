# 旧版总览

## 定位

这个分支保留的是迁移到 CloudBase 之前的旧版 Task Mate 实现。

它采用的是拆分式架构：

- Taro + React 客户端
- Express 服务端
- Mongoose + MongoDB
- WebSocket 驱动的面板刷新

## 当时它的价值

- 一个后端可以支持多端目标
- 任务逻辑和 AI 调用集中在传统服务端中
- WebSocket 可以减少任务变化后的全量轮询
- MongoDB 在早期产品迭代阶段有较高灵活度

## 被归档的原因

后来改用 CloudBase 原生方案，主要原因是：

1. 以当前产品范围来说，云函数方案成本更低、维护更轻
2. 它直接匹配微信小程序运行时
3. 不再需要单独托管 API 服务
4. 不再需要单独维护 WebSocket 实时通道
5. 当前产品方向明确偏向微信专属，因此多端和 Web 暂时不再是近期重点

## 旧版技术栈

### 客户端

- Taro 4
- React 18
- Sass
- 基于 REST 的任务调用
- 基于 WebSocket 的面板刷新

### 服务端

- Express 5
- Mongoose + MongoDB
- 围绕微信登录的鉴权流程
- 可选的 OpenAI 辅助能力
- 订阅消息相关定时逻辑

## 这个分支现在还保留的价值

- 旧任务控制器与状态语义
- WebSocket 刷新设计
- 早期前后端拆分边界
- 历史数据迁移前提
- 旧 API 路由结构

## 不再适合作为主线的部分

- 这个分支不再是当前部署目标
- 不应继续作为现行架构参考
- 除非是为了历史追溯，否则新功能应继续在 CloudBase 主线上推进
