# 发布与回滚

## 发布顺序

1. 确认分支和工作区。
2. 确认 `miniprogram/config/cloud.js` 指向目标环境。
3. 上传生产云函数。
4. 复核环境变量。
5. 复核数据库集合和索引。
6. 重新编译并上传小程序。
7. 按 `integration-checklist_cn.md` 做冒烟验证。

## 回滚策略

### 前端回滚

- 切回上一个稳定提交
- 重新编译并上传小程序

### 云函数回滚

- 从 Git 上传上一个稳定版本函数代码
- 再次检查环境变量

### 环境回滚

- 将小程序切回旧环境
- 保持旧环境继续服务，直到新环境修复
